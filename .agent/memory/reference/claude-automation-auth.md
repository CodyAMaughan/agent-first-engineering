# reference/claude-automation-auth
Running Claude **unattended** (webhook/cron/CI/ticket-driven): the supported path is the **Anthropic API key** (Agent SDK, Managed Agents, or `claude -p --bare`). The Claude Agent SDK **forbids** claude.ai/subscription login for third-party agents — API key only. A **Max subscription** works only for *first-party* Claude Code flows: the GitHub Action and non-bare `claude -p` via `claude setup-token` → `CLAUDE_CODE_OAUTH_TOKEN`. From **2026-06-15**, subscription Agent-SDK / `claude -p` / Action usage draws a small monthly **Agent SDK credit** (Pro $20 / Max-5x $100 / Max-20x $200) then bills at **standard API rates**; subscription rate limits (5-hour + weekly, shared with interactive use) make bursty backends impractical. Net: subscription for interactive/low-volume, API key for the always-on backend.
Sources: code.claude.com/docs/en/agent-sdk/overview · code.claude.com/docs/en/authentication · support.claude.com/en/articles/15036540
