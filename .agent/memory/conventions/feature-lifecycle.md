# conventions/feature-lifecycle
`feature-pipeline` (`.claude/workflows/feature-pipeline.js`) + the `feature-lifecycle` skill drive a feature request → review-ready branch through looping stages (spec → plan → implement → test-gate → review), each gated by a reviewer subagent or `TEST_CMD`, stopping before the PR for a human. Stack config lives in `.agent/lifecycle.conf` (`TEST_CMD`, branch naming, per-stage caps). Workflow `agent()` calls run as **subagents on the session's Claude account** (not a separate API key); for unattended use, port the same loop to the **Agent SDK with an API key**.
